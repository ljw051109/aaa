# 药品仓储管理系统
本仓库所储存的项目为药品仓储管理系统，系统整个代码体系通过文件读写实现数据持久化，采用分层架构保障代码规范性，核心功能覆盖用户管理、药品管理、库存变动等仓储核心场景，代码逻辑清晰、可扩展性强。

项目基本信息

项目名称：药品仓储管理系统

开发语言：java

开发环境：JDK1.8+，IDEA

运行环境：Windows，macOS，Linux


项目简介
    
    本药品仓储管理系统是一款基于Java Swing开发的桌面端应用，面向药品仓储管理的实际业务场景，实现了用户登录注册、药品信息管理、药品入库、药品出库、药品搜索等核心功能。系统采用分层设计思想，分为 Dao层，entity层，service层，ui层。各层职责清晰，低耦合，保障了系统的可维护性和延展性。

环境配置

1. 开发环境配置
安装 JDK 1.8 及以上版本，配置 Java 环境变量（JAVA_HOME、Path、CLASSPATH）；
下载并安装 Java 开发工具，推荐 IntelliJ IDEA 或 Eclipse；
将项目导入开发工具，确认项目 SDK 为已安装的 JDK 版本。
2. 运行环境配置
确保运行设备已安装 JRE 1.8 及以上版本；
直接运行项目编译后的 class 文件，或打包为 JAR 包后通过java -jar 包名.jar命令运行。

运行步骤

项目克隆：将本 GitHub 仓库代码克隆到本地，命令如下：plaintextgit clone [本项目GitHub仓库链接]
项目导入：打开 Java 开发工具，选择「导入项目」，找到克隆后的项目目录，完成导入；
初始化数据：项目data目录下已包含空的user.txt和products.txt，首次运行自动创建数据记录，也可手动添加测试数据；
启动项目：运行com.smart.ui.Login类中的main方法，启动系统，进入登录界面；

系统操作：

无账号用户先进行注册，填写用户名、密码、性别、地址、电话等信息，完成后自动跳转至登录界面；
已有账号用户直接输入用户名和密码，登录系统进入主界面；
在主界面可进行药品搜索、新增药品、药品入库、药品出库、删除药品等操作。

核心功能说明

1. 用户模块
用户注册：生成随机用户 ID，收集用户用户名、密码、性别、地址、电话信息，持久化存储到user.txt；
用户登录：校验用户名和密码的有效性，验证通过后进入系统主界面，验证失败则提示错误信息；
用户信息管理：通过 Dao 层和 Service 层实现用户信息的增、删、改、查，为系统操作提供身份验证基础。
2. 药品信息管理模块
新增药品：录入药品编号、名称、仓库分区类型、单价、入库数量等信息，默认仓储状态为「在库」，数据持久化到products.txt；
药品展示：以表格形式展示所有药品的编号、名称、类型、单价、库存数量、仓储状态，直观呈现库存信息；
药品删除：选择表格中的指定药品，确认后删除该药品的所有数据记录，同步更新数据文件；
药品搜索：根据「仓库分区类型」进行模糊搜索，快速筛选目标药品，提升数据检索效率。
3. 库存操作模块
药品入库：选择指定药品，输入入库数量，系统自动累加库存数量，更新药品数据记录；
药品出库：选择指定药品，输入出库数量，系统自动扣减库存数量（库存数量最低为 0），同步更新药品数据记录；
库存实时监控：所有出入库操作实时反馈到药品表格，实现库存数量的动态更新，便于仓储管理人员掌握库存状态。

核心技术与分层设计

1. 核心技术
Java Swing：实现桌面可视化界面开发，包含窗体、表格、按钮、文本框、下拉框等组件，完成界面布局和事件监听；
文件 IO 流：采用 BufferedReader、BufferedWriter 实现文本文件的高效读写，完成用户和药品数据的持久化存储；
面向对象编程：运用封装、继承、多态思想，定义实体类和业务类，提高代码的复用性和可维护性；
事件驱动模型：为按钮、文本框等组件添加事件监听器，实现界面操作与业务逻辑的联动。
2. 分层设计说明
本项目采用四层架构设计，各层职责单一、相互解耦，符合软件设计的单一职责原则和低耦合高内聚原则，具体说明如下：
表格层级核心类主要职责Dao 层UserDao、ProductDao仅负责数据访问，实现与文本文件的交互，提供数据的读、写、增、删等基础方法；entity 层User、Product定义系统核心实体，封装属性和 get/set 方法、toString 方法，作为数据传输载体；service 层UserService处理业务逻辑，调用 Dao 层的方法，封装用户信息的增删改查业务，为 UI 层提供接口；ui 层Login、Register、Home实现可视化界面，处理用户界面操作事件，调用 Service 层方法完成业务逻辑，展示运行结果；

开发亮点

分层架构设计：四层架构分工明确，代码结构清晰，便于后续功能扩展和代码维护，例如新增「药品效期管理」功能仅需在对应层级添加代码，不影响其他模块；
数据持久化：采用文件 IO 实现数据本地存储，无需数据库环境，降低系统运行门槛，适合中小型仓储场景的快速部署和使用；
操作便捷性：可视化界面简洁友好，所有操作通过按钮、表格、弹窗完成，无需专业的计算机知识，仓储管理人员可快速上手；
数据校验与异常处理：对出入库数量、用户输入等进行基础校验，添加 IO 异常捕获，避免因用户误操作或文件异常导致系统崩溃；
功能贴合实际：基于药品仓储管理的实际业务需求开发，覆盖从用户身份验证到药品全生命周期的仓储管理，满足 GSP 认证对药品仓储可追溯的基本要求。

开发中遇到的问题与解决方法

问题1：多次点击新增药品按钮导致重复创建弹窗
现象：快速多次点击「新增商品」按钮，会创建多个新增药品弹窗，造成操作混乱；解决方法：将新增药品弹窗设置为模态对话框（Modal），打开弹窗后屏蔽主界面的操作，关闭弹窗后再恢复主界面交互，避免重复创建。

问题2：删除药品时未选择行导致的空指针异常
现象：未在表格中选择任何药品，直接点击「删除商品」按钮，系统抛出空指针异常；解决方法：在删除方法中先判断表格是否选中行，若未选中则弹窗提示「请选择商品」，直接返回方法，避免执行后续的删除逻辑。
